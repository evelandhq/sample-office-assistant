# 办公流程与材料清单

当用户的问题重点是“怎么做”“要准备什么”“找谁审批”“有没有时限/额度/前提条件”时，使用本 skill。

这个 skill 和“办公申请与说明撰写”不同：

- 本 skill 负责把政策整理成步骤、条件、材料、角色和限制。
- 撰写 skill 负责把已知信息写成邮件、消息、申请或说明。

如果用户同时要求“先告诉我流程，再帮我写”，可以先按本 skill 整理规则，再在答案后半段切换到撰写产物。

## 必须遵守的前提

- 只能依据 `/workspace/office-knowledge/` 中实际读取到的内容回答。
- 不得凭常识补充审批层级、金额上限、办理时限、例外条件或需要提交的材料。
- 如果知识库没有足够资料支撑某一步骤或字段，必须明确指出缺口，并提醒用户联系行政助理小琳补充。
- 最终答案仍需保留主指令要求的免责声明与来源标注。

## 适用任务

- 请假前要准备什么
- 漏打卡或迟到后怎么处理
- 远程办公需要哪些审批或备案
- 报销和出差分别需要哪些材料与限制
- 访客、工位、会议室、门禁、安全事件怎么走流程

## 工作步骤

1. 判断用户要的是哪种结构：步骤清单、材料清单、审批链、对比表、是否可行判断。
2. 提取问题中的关键约束：事项类型、日期、地点、金额、是否紧急、是否已有审批、是否跨地区。
3. 先检索再阅读命中的知识库段落，确认：
   - 适用范围
   - 触发条件
   - 审批或备案角色
   - 时间要求
   - 金额或次数限制
   - 必备材料和例外
4. 用最直接的结构输出，不要把流程信息写成大段散文。
5. 如果用户信息不足以判断适用路径，先列出需要补充的关键条件，再给出基于现有信息的条件化答案。

## 输出格式要求

- 第一段先给结论，例如“按当前示例政策，你需要先完成审批，再补齐发票与行程材料。”
- 然后优先使用以下结构之一：
  - 步骤清单
  - 材料清单
  - 审批链
  - 条件对比表
- 如果政策里有例外、限制或冲突条款，单独列出“注意事项”。
- 如果知识库资料不足，直接写：`当前知识库没有足够资料回答这个问题。` 然后提醒联系行政助理小琳。

## 推荐骨架

可参考同目录下的 `references/checklist-patterns.md` 来组织答案结构，但实际输出只能保留本次知识库命中的内容。
