# 办公申请与说明撰写

当用户要求你“帮我写”“起草一下”“给个模板”“整理成邮件/消息/申请单”这类内容时，使用本 skill。

你的目标不是凭空写通用套话，而是把知识库里的示例政策，整理成用户可直接改写或提交的文本。

## 必须遵守的前提

- 先按主指令要求检索 `/workspace/office-knowledge/`，确认相关政策后再起草。
- 不得用常识补齐政策事实，不得编造额度、时限、审批人、流程或例外。
- 如果知识库没有足够资料支撑某个字段或结论，就明确写出缺失点，并提醒用户联系行政助理小琳补充。
- 起草内容仍然属于“示例公司政策”场景，最终答案里保留主指令要求的免责声明与来源。

## 适用任务

- 请假申请
- 远程办公申请
- 考勤异常说明
- 报销说明或出差申请
- 办公场所事件说明

## 工作步骤

1. 先判断用户要的文体：邮件、IM 消息、正式申请、说明、清单、表格字段。
2. 从用户输入中提取已知信息：事由、日期、时段、地点、金额、附件、审批对象、紧急程度。
3. 检索并阅读对应政策段落，确认：
   - 是否允许该事项
   - 需要哪些前置条件
   - 要找谁审批或备案
   - 是否有时间限制、金额限制、材料要求
4. 如果起草所需关键信息缺失，先列出缺口，再给一个带占位符的版本。
5. 按用户指定文体输出最终草稿，并尽量让文本可直接复制使用。

## 输出格式要求

- 先给一句简短结论，例如“可以按以下示例邮件提交”或“当前知识库不足以支持完整报销说明”。
- 然后输出成稿；如果需要，也可补一个“需要你补充的信息”小清单。
- 对模板中的可替换内容，用全角方括号占位，例如 `【请假日期】`、`【直属经理姓名】`。
- 若知识库支持多个步骤或材料，优先把它们嵌入正文；只有在正文不合适时才单列清单。

## 推荐模板骨架

可参考同目录下的 `references/templates.md`，但只能保留与本次知识库命中内容一致的字段，不能机械照抄。
