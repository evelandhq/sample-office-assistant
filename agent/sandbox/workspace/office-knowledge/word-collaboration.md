# Word 文档协作与审阅策略

## 适用场景

适用于多人共同起草 Word 文档、收集意见、控制编辑权限，以及通过评论和修订形成可审阅的定稿过程。

## 核心策略

### 维护单一共享版本

1. 将文档保存在 OneDrive 或 SharePoint Online，并发送共享链接，避免邮件附件产生多个副本。
2. 根据角色授予编辑、审阅或仅查看权限。只有需要直接改写正文的人才授予编辑权限。
3. 支持的 Word 版本可以实时共同编辑；较旧版本可能需要保存或刷新后才能看到他人更改。
4. 保存共享文档会把本地更改写回服务器、刷新其他作者的更改，并更新服务器上的版本历史。

### 区分评论与修订

1. 评论用于提问、解释和建议，不直接改变正文；选择具体文字后添加评论，使反馈有明确上下文。
2. 使用 `@` 提及把某条评论通知给指定作者；回复形成可追踪的讨论线程。
3. 修订模式用于提出可逐项接受或拒绝的文字改动。需要强制审阅流程时，可在共享时启用仅以审阅模式打开，但该共享能力主要适用于 Word 网页版。
4. 已解决的评论仍保留在文档中，便于定稿前追溯；删除评论是永久操作，适合在确认完成后再进行。

### 定稿检查

1. 逐项接受或拒绝修订，不要仅隐藏修订标记就当作已经定稿。
2. 确认所有关键评论已经回复或解决，再清理不需保留的讨论。
3. 必要时使用“比较”功能核对版本差异；如共同编辑不可用，先检查文件位置、Word 版本和文件格式。
4. Word 网页版共同编辑通常需要 `.docx`；旧 `.doc` 文件应先转换，并保留原文件备份。
